# Frontend Design

Use this skill when building or redesigning UI and you need intentional design decisions, not template defaults.

## Design Baseline

- Define the product context first: user type, primary workflow, and page goal.
- Make one clear visual thesis in the first viewport.
- Use style choices that match the domain, not generic showcase layouts.

## Workflow

1. Define constraints:
   - Target device mix, accessibility expectations, and data density needs.
2. Build a compact design system:
   - 4-6 core colors, semantic states, and consistent spacing scale.
   - Distinct typography roles (display, body, utility).
3. Plan structure:
   - Prioritize scanability, repeated actions, and workflow continuity.
4. Implement interaction:
   - Clear affordances, keyboard focus, and reduced-motion support.
5. Critique and adjust:
   - Verify text fit, hierarchy, spacing stability, and no overlapping UI.

## Content Rules

- Prefer concrete product language over vague marketing copy.
- Use labels that match user actions and expected outcomes.
- Keep empty and error states actionable.

## Quality Bar

- Responsive behavior across mobile and desktop.
- Accessibility and focus states are visible and usable.
- Visual style is consistent and not dominated by one generic theme.
